# Limpieza Semanal de Issues

## Instrucciones:

1. Issues sin actividad > 14 días:
   - Si están "In Progress" → agregar comentario pidiendo actualización
   - Si están "In Review" → verificar PR asociado
   
2. Issues sin asignar en "Todo":
   - Distribuir equitativamente entre el equipo
   - Priorizar por antigüedad
   
3. Issues completados sin PR vinculado:
   - Agregar etiqueta "needs-verification"
   
4. Backlog:
   - Archivar issues > 6 meses sin actividad
   - Re-priorizar según nuevos criterios
   
5. Generar reporte de acciones tomadas

Solicitar confirmación antes de aplicar cambios masivos.
